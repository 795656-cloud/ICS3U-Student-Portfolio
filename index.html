Distributed File Storage System

A scalable distributed system for storing and retrieving files with built-in redundancy and fault tolerance

Overview

Architecture Diagram Placeholder

This project implements a distributed file storage system inspired by Google File System (GFS). It supports storing large files across multiple servers with automatic replication for fault tolerance and efficient data retrieval.

Technologies Used

Go gRPC Docker Kubernetes Redis

Key Features

  • Automatic file chunking and distribution across multiple servers
  • Three-way replication for data redundancy
  • Master-slave architecture for metadata management
  • Heartbeat mechanism for detecting node failures
  • Automatic re-replication when nodes fail
  • Load balancing for read operations

Technical Highlights

The system uses consistent hashing to distribute files evenly across storage nodes. I implemented a custom consensus protocol for handling master node failures and ensuring metadata consistency. The system can handle concurrent reads and writes while maintaining data integrity.
